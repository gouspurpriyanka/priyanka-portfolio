Artifact 2

EatWise AI Help Proxy

Deployment

A live Next.js deployment that brands Chatbase help content into a product-style EatWise AI support experience with user-friendly design and seamless routing.

Overview: EatWise AI Help Proxy is a deployed Next.js application that provides a branded entry point for the EatWise AI help experience. It connects users to a Chatbase-hosted help page through a Vercel deployment.

Why I Created It: I created this artifact as part of my AI Lab work to show how an AI support resource can be packaged into a professional, user-facing web experience. Instead of only linking directly to an external tool, I designed a simple branded page that makes the help center feel connected to the EatWise AI product.

What I Learned: This project helped me understand how AI tools can be integrated into a web application workflow. I practiced configuring environment variables, using Next.js rewrites, deploying to Vercel, and troubleshooting production issues such as missing environment values and server errors.

How I Built It: I built a minimal Next.js app, configured a /help route to connect with the Chatbase help page, added a polished EatWise AI landing page, and deployed the finished project to Vercel for public access.

Skills Demonstrated: AI Tool Integration • Web Deployment • Next.js • Vercel Hosting • Debugging • User Experience Design

Professional Relevance: This artifact demonstrates my ability to move an AI lab activity beyond experimentation and into a usable product experience. It shows that I can connect AI services, web development, deployment, and user-centered design.

AI Lab Reflection

Why This AI Lab Project Belongs in My Portfolio

I selected the EatWise AI Help Proxy as an artifact because it shows the kind of practical AI implementation work I want to continue developing. The project was small in scope, but it required the same thinking used in professional software work: understanding the user need, connecting an AI service, designing a clean interface, deploying the result, and testing it in production.

This experience also helped me see that AI work is not only about building models. It is also about making AI tools accessible, trustworthy, and easy for people to use. By turning a Chatbase help page into a branded EatWise AI support experience, I practiced presenting AI in a way that feels more complete and useful for an audience.

Adding this artifact to my portfolio gives evidence of my ability to learn quickly, troubleshoot technical problems, and create a working AI-connected web project. It reflects my interest in applying AI to real user support and product experiences.
